# Immortal Path: Idle Loot

UE 5.7 + C++ 开发的 2D 横版挂机刷怪 ARPG。

## 当前开发进度

### 第 1 步：玩家自动攻击核心（已完成）

现有 `Content/GAME/Player/Player` 蓝图已改为继承 `ImmortalPlayerCharacter`，并已在 UE 5.7 编辑器中编译、保存和确认战斗参数可见。

C++ 已实现：

- 周期性搜索攻击范围内的目标
- 优先选择距离玩家最近的目标
- 攻击前摇结束后通过 UE `ApplyDamage` 统一结算伤害
- 目标离开范围或失效时不造成伤害
- 可在蓝图中配置伤害、攻击范围、攻击间隔和前摇
- 为攻击动画提供 `On Auto Attack Started` 蓝图事件
- 为命中反馈/恢复待机提供 `On Auto Attack Resolved` 蓝图事件
- 临时蓝图目标只需带 `Monster` Actor Tag 即可被攻击

主要代码：

- `Source/ImmortalPath/Characters/ImmortalPlayerCharacter.h/.cpp`
- `Source/ImmortalPath/Combat/AutoAttackTarget.h`

### 将现有 Player 蓝图接入 C++

接入已经完成。如需复查，打开 `Content/GAME/Player/Player`，顶部应显示父类 `ImmortalPlayerCharacter`；在 Class Defaults 的 `Immortal Path | Combat` 分类下可以调整参数。

默认参数：

- Attack Damage：10
- Attack Range：450 cm
- Attack Interval：1 秒
- Attack Windup：0.25 秒

### 临时测试方法

在下一步的正式怪物类完成前，可以给一个带碰撞的 Pawn/Actor 添加 `Monster` Actor Tag，碰撞对象类型使用 Pawn 或 WorldDynamic，并在它的 `AnyDamage` 事件后接 `Print String`。把它放到玩家 220 cm 内即可验证自动攻击。

### 第 3 步：怪物自动战斗与动画（已完成）

### 第 6 步：玩家/怪物血条与装备光球（已完成）

已导入并接入用户提供的分层 UI 素材：

- 玩家状态条：`512×64`，由背景、生命填充、法力填充和边框组成
- 怪物头顶血条：`256×32`，由背景、生命填充和边框组成
- 通用装备光球：`128×128` 透明纹理

C++ 已实现：

- 游戏开始时自动创建玩家 HUD，显示当前生命和法力百分比
- 每只 Fox/Dog 自动创建屏幕空间头顶血条，实时读取怪物生命百分比
- 怪物死亡时按照各自 `EquipmentDropChance` 概率生成装备光球 Actor
- 光球短暂漂浮后自动追踪玩家，无需玩家主动移动或按键拾取
- 拾取后增加玩家的待入库装备数量，并触发 `On Equipment Picked Up` 蓝图事件
- 光球触发 `On Equipment Collected` 事件，预留拾取音效和粒子效果

运行验证：

- PIE 中玩家血/蓝条和怪物头顶血条均正确显示
- 强制生成光球后成功自动拾取，玩家装备计数从 0 增加到 1
- `ImmortalPath` 和 `ImmortalPathEditor` 的 Win64 Development 目标均编译通过

### 第 7 步：装备数据、背包与自动换装（已完成）

装备数据结构：

- 五个部位：武器、头部、衣服、鞋子、饰品
- 五档品质：普通、优秀、稀有、史诗、传说
- 随机属性：攻击、防御、生命、攻击速度和暴击率
- 每件装备具有唯一 ID、装备等级、中文名称和独立战力评分
- 光球会根据品质显示白、绿、蓝、紫、金色

背包与装备规则：

- 玩家默认拥有 30 格背包和五个装备槽位
- 拾取装备后自动与同部位当前装备比较战力
- 更强装备自动穿戴，被替换装备进入背包
- 较弱装备直接进入背包；背包满时只保留比当前最弱物品更强的装备
- 已装备属性实时加入玩家最大生命、攻击、防御、攻速和暴击率
- 攻速变化后自动刷新挂机攻击计时器
- 提供背包、装备栏、品质颜色、自动换装和战力的蓝图查询/事件接口

运行验证：

- 连续生成并拾取 8 件随机装备，装备计数正确记录为 8
- 自动穿戴稀有衣服、史诗武器、鞋子和饰品，4 件旧/较弱装备进入背包
- 角色综合战力由基础值提升至约 169.61
- 普通、稀有和史诗装备均在一次测试循环中成功生成

### 第 8 步：正式背包与装备界面（已完成）

- 已导入用户提供的 15 个背包 UI 资源：900×600 面板、关闭按钮、五部位图标、五品质框和三种槽位状态
- 玩家状态条右侧新增“背包 [I]”按钮，也可以直接按 `I` 键打开或关闭
- 左侧显示武器、头部、衣服、鞋子和饰品五个装备位；空槽显示半透明部位提示
- 右侧显示 5×6、共 30 格背包，支持滚动和已使用容量显示
- 装备格显示部位图标、品质边框、装备等级、普通/选中/已装备状态
- 选择装备后显示名称、部位、等级、攻击、防御、生命、攻速、暴击和装备战力
- 背包装备会与同部位当前装备比较，使用绿色/红色显示战力差；已穿戴装备显示“已装备”
- 顶部实时显示角色综合战斗力，装备拾取和自动换装后界面自动刷新
- 界面打开时挂机战斗继续运行，鼠标可操作；关闭后恢复纯游戏输入
- 面板会根据实际视口尺寸居中：1280×720 验证结果为 900×600 面板位于 `(190, 60)`

运行验证：

- 连续生成并拾取 15 件随机装备后，4 个部位成功自动穿戴，12 件装备进入背包
- 普通、优秀、稀有、史诗品质均正确参与自动比较和换装
- 角色综合战斗力从 108.00 提升至 173.74
- `ImmortalPath` 与 `ImmortalPathEditor` Win64 Development 目标均编译通过
